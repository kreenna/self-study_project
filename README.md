# Платформа самообучения студентов

## Описание проекта

Платформа для самообучения студентов, которая позволяет регистрироваться и авторизовываться пользователям, создавать и управлять обучающими материалами и разделами, а также проходить тесты для проверки знаний. Все основные сущности доступны для управления через административную панель Django и REST API.

## Структура проекта
```
project_root/
├── manage.py
├── config/ # Основной конфигурационный пакет (settings, urls)
├── users/ # Приложение пользователей (авторизация, роли, права)
├── materials/ # Приложение курсов, уроков, тестирования
├── requirements.txt # Файл зависимостей (можно сгенерировать из poetry)
├── README.md # Этот файл
└── ... # другие папки и файлы
```

## Используемые технологии

- Python 3.13;
- Django;
- Django REST Framework (DRF);
- PostgreSQL;
- JWT (djangorestframework-simplejwt) для аутентификации;
- drf-yasg для автодокументации API (Swagger);
- django-cors-headers для CORS;
- Poetry для управления зависимостями (опционально)

## Установка и запуск

1. Клонировать репозиторий:
```
git clone https://github.com/yourusername/yourproject.git
cd self-study_project
```

2. Создать виртуальное окружение:
```
python -m venv venv
source venv/bin/activate # или venv\Scripts\activate на Windows
```

3. Установить зависимости через Poetry или pip:

- Если используете Poetry:
```
poetry install
```

- Если pip, то:
```
pip install -r requirements.txt
```

4. Настроить файл `.env` с настройками базы данных, секрета и т.п
```
POSTGRES_DB=self-
POSTGRES_USER=
POSTGRES_PASSWORD=
POSTGRES_HOST=
POSTGRES_PORT=
```

6. Сделать миграции и применить их:
```
python manage.py migrate
```

6. Запустить сервер разработки:
```
python manage.py runserver
```

## API документация

После запуска сервера доступна документация API:

- Swagger UI: `http://127.0.0.1:8000/swagger/`
- Redoc: `http://127.0.0.1:8000/redoc/`

## Функционал платформы

- Регистрация и вход пользователей с JWT авторизацией
- CRUD операции над курсами, уроками и тестами через API и Django Admin
- Проверка и сдача тестов отдельным запросом
- Система ролей и прав:
  - Администраторы: полный доступ
  - Владельцы курсов (Преподаватели): управление своими курсами и тестами
  - Студенты: просмотр материалов и прохождение тестов

## Тестирование
